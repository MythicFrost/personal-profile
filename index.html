Project Idea

Overview

My project idea is an app to supplement bible study. Initially the project would begin as a web app that works on both desktop and smartphone; following that, an app for Android and then for iOS would be released.

There are many books in the bible, with many chapters and verses across a vast range of topics; this app's purpose is to make it easy to keep lists of scriptures sorted by user created tags (topics.) It also adds an aspect of community to it as the user can see what tags and scriptures that others have added.

The app will display a screen of tags (E.G., "law", "mercy", "love", "messianic prophecy", "dietary laws") that have been added by both themselves and other users. When clicking on a tag (for example "love") it will show all the scriptures that have had that tag added to them.

The app will also display a screen where the user can look up scriptures and apply tags that they've created.

Motivation

Christianity is the largest religious group in the world with 2.3 billion believers as of 2015. (Hackett and Mcclendon, 2017)

This app may appeal to those who are interested in improving their knowledge of the bible, debating or studying specific topics, or even just randomly exploring other topics with which they may not be very familiar.

I myself read the bible very often and would appreciate being able to use this app to keep track of scriptures on particular topics for both debating and studying purposes.

Description

As stated above, the app allows users to create their own tags which they can then apply to various scriptures. They can then view all the scriptures associated with a given tag.

The user can browse tags and scriptures added within the community, or just their own tags and scriptures that they've added.

There is no need to sign up to use the app - a user can view community tags and tagged scriptures; The benefits to signing up include being able to create your own tags, to tag scriptures, and to read a translation of the scriptures (E.G., the NIV).

Bible related ads at the top of the app will cover necessary hosting costs. There is no fee for the user to sign up; however, ads could be removed with a monthly subscription.

Additional considerations that could be taken into account are additional bible translations, support for different languages, a discussion forum or "prayer request" submissions section.

The app(s) would include functionality to send you an email (for the web app) or a push notification (for the Android and iOS apps) on the verses that you have tagged.

Licensing fees and hosting fees are potential difficulties that may need to be addressed.

Some key features:

  • Users can quickly add or merge community tags and scriptures to their own.
  • Users can upvote or downvote tags based on their relevance to the related scriptures, improving accuracy. (This would not affect a user's personal tags, only community tags).
  • Users can create a list of scriptures which is immutable to all but the author; the list should describe a narrative on a topic. These lists could be randomly featured on the site, allowing for upvotes to indicate interesting content.

Tools and Technologies

For the web app, I would utilise a front-end framework such as Angular and a server-side language such as PHP. For the database I would use MySQL. Additional technologies (or alternatives) might include Typescript, Node.js, React, or Vue.

Visual Studio Code and Brackets are great editors for many of these languages. For building the Android app I would use Java in Android Studio; for iOS I would use Swift in XCode.

A suitable ads framework for each platform (web, iOS, Android) would be required.

I already have the necessary devices at my disposal for development & testing: Windows PC, Mac, iPhone and Android smartphone.

Required Skills

An assortment of programming skills will be required to produce the apps, including: HTML, CSS, Javascript, PHP, Javascript frameworks, MySQL, Java, Swift.

Also, the data architecture will be incredibly important as the app(s) will need to categorise and structure the tags, scriptures, and their associations efficiently in a database.

I already possess many of these core skills, however I am not familiar with the aforementioned frameworks for the front-end, so I will need to follow some tutorials and courses.

I will also need to liaise with companies to procure licenses for their bible translations. I have already contacted the company responsible for licensing the NIV.

Outcome

If the project is successful, it will provide an easy and fun way for users to learn and keep track of scriptures on a variety of different topics. It has potential to expand in the future to other languages or additional versions of the bible.

An additional benefit is forming a community of people interested in bible study whom are actively contributing to the site just by using it; you may stumble across community tags for different topics you've not studied before, or see scriptures you haven't read before that are related to topics you're interested in.

The project would also demonstrate my capabilities; it would be a great item to have in my portfolio.
